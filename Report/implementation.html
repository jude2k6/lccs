Website

When researching what would be needed to make an interactive website with a python backend 3 frameworks popped up Django, Fastapi and Flask. After further research Flask became the obvious choice as it was the one most suited to small scale apps where performance is not a major factor.

After setting up the file system seen in figure 3.4 I began to write the code that would serve the website and establish all of the paths for the different pages on the website that would be accessed through the navigation bar, this was done in views.py

User recommendations

To make recommendations to the user I had to take an input off of the user and send it to my flask backend thankfully this is made easy by flask using post methods. When a user presses the submit button on the form a post request is sent to the back end with the data packaged as a dictionary. I then filter my cleaned data based on what the user submitted and then return the average salary of the people that meet this criteria and the percentage more or less that the user makes.
